# 评分模型

本文档说明 `bp_engine.py` 中推荐分的主要组成。这里描述的是内部排序模型，不是真实胜率，也不是 Lolalytics 原始胜率。


## BP 状态

推荐接口使用四个下路槽位：

```python
{
    "ally_ad": None,
    "ally_sup": None,
    "enemy_ad": None,
    "enemy_sup": None,
}
```

状态推导：

- S1：双方下路未知，主要看基础表现。
- S2：已知己方搭档，优先看配合。
- S3：只知道敌方 ADC，优先看对 ADC 的克制。
- S4：只知道敌方辅助，优先看对辅助的克制。
- S5：已知敌方下路组合，综合双人对位和基础表现。
- S6：己方搭档 + 敌方 ADC，综合配合和克制。
- S7：己方搭档 + 敌方辅助，综合配合和克制。
- S8：己方搭档和敌方下路组合都已知，综合基础表现、配合和对位。


## 总分

```text
原始推荐分 = 基础校准分 × 基础权重 + 配合校准分 × 配合权重 + 对位校准分 × 对位权重
最终推荐分 = 原始推荐分 - 语境惩罚
推荐指数 = 100 / (1 + 10^(-最终推荐分 / 400))
```

推荐指数只用于排序和展示，不应解读为胜率。


## 动态权重

当前权重由 BP 状态决定：

```text
S1 盲选：基础 100%
只知道己方搭档：基础 35%，配合 65%
只知道敌方信息：基础 40%，对位 60%
己方搭档 + 敌方信息：基础 20%，配合 40%，对位 40%
```

如果某类信息不可用，权重会归一化到可用部分。


## 基础表现

基础表现由 Tier 先验和英雄自身胜率修正组成：

```text
基础表现 = Tier 先验 + 胜率修正
胜率修正 = winrateToRating(平滑英雄胜率) × min(1, sqrt(英雄场次 / 3000))
平滑英雄胜率 = (胜场 + 1000 × 50%) / (场次 + 1000)
基础校准分 = 22 × tanh(基础表现 / 30)
```

设计意图：

- 低场次英雄不因为短期高胜率被过度放大。
- Tier 只作为基础表现先验，不重复参与配合和对位的期望值。


## 配合分

配合分衡量候选英雄和己方下路搭档的组合表现。

```text
配合分 = 置信度 × (45% × 组合绝对表现 + 35% × 超预期配合 + 20% × 组合成熟度)
组合绝对表现 = winrateToRating(组合胜率相对 50% 的表现)
超预期配合 = 组合实际 rating - 双方期望 rating
配合校准分 = 30 × tanh(配合修正分 / 30)
```

有己方搭档但缺少配合样本时，模型会做较强保守惩罚，避免候选只靠版本强度进入高位推荐。


## 对位分

对位分衡量候选英雄面对敌方 ADC 和/或辅助时的表现。

```text
对位分 = 置信度 × (45% × 对位绝对表现 + 35% × 超预期对位 + 20% × 对位样本成熟度)
超预期对位 = 实际对位 rating - 理论对位 rating
对位校准分 = 30 × tanh(对位修正分 / 30)
```

推荐辅助时更重视敌方辅助对位，推荐 ADC 时更重视敌方 ADC 对位。有敌方信息但缺对位样本时，会按缺失比例做轻微保守惩罚。


## 语境惩罚

语境惩罚用于处理“版本强势英雄很强，但当前 BP 并不支持它”的情况。英雄可以因为基础强度高而排在前面，但如果配合和对位都没有给到足够理由，它不应无条件压过更适合当前局面的选择。

```text
只知道己方搭档：语境分 = 配合校准分
只知道敌方信息：语境分 = 对位校准分
己方搭档 + 敌方信息：语境分 = (配合校准分 + 对位校准分) / 2
盲选：不计算语境惩罚

语境差距 = 基础校准分 - 语境分
如果语境差距 > 6：
差距惩罚 = min(4, (语境差距 - 6) × 0.25)

如果基础校准分 > 14 且语境分 < 13：
Top 优先级惩罚 = (13 - 语境分) × 0.45 + (基础校准分 - 14) × 0.12

语境惩罚 = min(5, 差距惩罚 + Top 优先级惩罚)
最终推荐分 = 原始推荐分 - 语境惩罚
```

设计意图：

- Smolder、Seraphine 这类版本强势英雄仍然可以获得较高基础分。
- 当它们和当前搭档/敌方组合没有明显适配时，会降低 Top1 优先级。
- 盲选场景没有搭档和敌方信息，因此不做语境惩罚。


## 成熟度与可信度

关系成熟度：

```text
全局成熟度 = min(10, log(1 + 场次) / log(1 + 3000) × 10)
英雄内成熟度 = min(10, 当前关系场次 / 该英雄最高关系场次 × 10)
成熟度 = max(全局成熟度, 0.85 × 英雄内成熟度)
实际绝对胜率低于 48.5% 时，成熟度按 0 处理
```

成熟关系托底：

```text
如果关系场次 ≥ 10000
且实际绝对胜率 ≥ 49%
且实际表现没有低于理论预期超过 1%
则该关系分最低为 3
```

返回给前端的结果还包含：

- `evidence_label`：推荐依据。
- `missing_labels`：缺失数据提示。
- `confidence_level` / `confidence_label`：样本可信度。
- `effective_weights`：当前英雄实际使用的权重。


## 数据处理原则

- 正式推荐过滤 `tier="?"` 的英雄。
- 已选英雄不会进入推荐结果。
- 缺失的 synergy/counter 数据不按 0 分当作中性优势。
- 组合/对位不会只看“是否超预期”，也会看实际胜率是否足够好。
- 高场次不会直接等于强，但会作为成熟度和样本稳定性进入配合、对位分。
