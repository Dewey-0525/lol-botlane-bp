# 微信小程序审核材料草稿

本文档用于后续提交微信小程序审核前整理名称、简介、功能说明和截图文案。


一、小程序基础信息
------------------

建议名称：

```text
下路 BP 助手
```

备选名称：

```text
LOL 下路 BP 助手
英雄联盟下路 BP 助手
下路组合推荐助手
```

一句话简介：

```text
基于公开对局统计数据的英雄联盟下路组合 BP 推荐工具。
```

详细简介：

```text
本工具用于英雄联盟下路 BP 场景，基于公开统计数据，为用户提供 ADC/辅助推荐、协同搭配查询、克制查询和梯队榜参考。推荐结果会展示数据依据、可信度和缺失数据提示，帮助用户更清楚地理解推荐来源。
```


二、核心功能说明
----------------

### 1. BP 推荐

用户可输入己方 ADC、己方辅助、敌方 ADC、敌方辅助，并选择推荐 ADC 或推荐辅助。系统根据当前 BP 信息自动判断状态，综合基础表现、下路配合和敌方对位生成推荐。

展示内容：

- 推荐英雄
- 梯队
- 推荐指数
- 基础表现
- 对位分
- 配合分
- 推荐依据
- 可信度
- 缺失数据提示


### 2. 协同查询

用户选择任意英雄后，可查询该英雄在下路组合中的高胜率搭档。


### 3. 克制查询

用户选择敌方英雄后，可查询推荐的 ADC 和辅助反制选择。


### 4. 梯队榜

用户可查看当前数据缓存中的 ADC 和辅助梯队榜。


三、数据来源说明
----------------

说明文案：

```text
本工具使用公开网站 Lolalytics 的英雄统计数据进行离线预热，后端读取本地缓存数据生成推荐。用户请求推荐时不会实时爬取第三方网站。
```

展示建议：

- 在“说明”页面展示数据来源。
- 在主页面展示数据更新时间。
- 明确推荐仅供参考。

五、隐私说明
------------

当前版本：

```text
不提供登录功能。
不收集微信昵称、头像、手机号、精确位置等个人信息。
用户在页面中选择的英雄仅用于本次推荐请求。
```

如果后续加入收藏、历史记录或登录功能，需要重新补充隐私说明。


六、截图准备
------------

建议准备 4 张截图：

1. BP 推荐页：展示四个 BP 槽位和推荐结果。
2. 协同查询页：展示某英雄最佳搭档。
3. 克制查询页：展示敌方英雄反制推荐。
4. 梯队榜页：展示 ADC 或辅助梯队。

截图内容建议：

- 使用真实但常见的示例，例如 Lucian / Jinx / Leona。
- 页面中应出现“可信度”“依据”“数据更新时间”等解释字段。
- 避免截图中出现错误、空数据或调试信息。


七、审核前检查清单
------------------

- 小程序页面可正常打开。
- 所有接口使用 HTTPS 域名。
- 微信后台已配置 request 合法域名。
- `/api/health` 正常。
- `/api/champions` 正常。
- `/api/recommend` 正常。
- 头像资源可正常加载。
- 说明页包含数据来源、免责声明、隐私说明。
- 页面中没有“测试”“调试”“localhost”“127.0.0.1”等正式环境不应出现的内容。


八、暂不申请的能力
------------------

当前版本不需要：

- 用户登录
- 获取手机号
- 获取位置
- 订阅消息
- 支付
- 客服消息

这样能降低审核复杂度。
